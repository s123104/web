安裝部署指引

就像搭建樂高積木一樣，我們將一步步建構你的 Claude Code 開發環境

系統需求檢查

macOS

macOS 10.15+ 原生支援，安裝最簡單

Linux

Ubuntu 18.04+ 或 CentOS 7+

Windows

需要 WSL2，Windows 10 2004+

1

準備 Node.js 環境

把 Node.js 想像成 Claude Code 的引擎，我們需要確保這個引擎版本夠新、效能夠好。 就像駕駛汽車前先檢查引擎一樣，這是最基礎也是最重要的步驟。

# 檢查 Node.js 版本（需要 18+）
node --version

如果版本低於 18，請前往 nodejs.org 下載最新穩定版

2

安裝 Claude Code

現在我們要安裝 Claude Code 本身。這個過程就像在電腦上安裝一個新的應用程式， npm 會自動處理所有相依性和設定，讓整個過程變得輕鬆簡單。

# 全域安裝 Claude Code
npm install -g @anthropic/claude

重要提醒：請避免使用 sudo npm install，這可能會造成權限問題。 如果遇到權限錯誤，建議設定 npm 使用使用者目錄。

3

認證設定

認證就像是給 Claude Code 一把鑰匙，讓它能夠代表你與 Anthropic 的服務溝通。 這個步驟建立了你與 AI 助手之間的信任橋樑。

驗證安裝

最後一步是確認所有元件都正常運作。就像新車交車時的最終檢查， 我們要確保每個功能都能正常使用。

使用規則與最佳實踐

就像學習開車需要了解交通規則一樣，掌握這些使用規則能讓你更有效地運用 Claude Code

💎 Claude Pro 使用限制詳解

時間額度制度

Claude Pro 採用「每 5 小時重置」的額度制度，就像電動車的充電循環。 你不需要等到用完才能重新獲得額度，而是每 5 小時自動獲得一次完整的使用額度。

實際範例：
上午 9:00 開始使用 → 下午 2:00 額度重置
下午 2:00 開始使用 → 晚上 7:00 額度重置

訊息數量計算

短對話至少可以發送 45 條訊息，但實際數量會根據訊息長度、檔案大小、 對話歷史等因素動態調整。想像成一個彈性的容器，內容越複雜，容量就會相對減少。

簡短對話：45+ 訊息
包含檔案：視檔案大小減少
長對話歷史：會影響可用額度

🚀 專業使用技巧

任務分割

將複雜專案拆分成小任務，每個對話專注處理一個具體問題，這樣既能節省額度，也能獲得更精準的結果

定期清理

適時使用 /clear 命令清除對話歷史，避免 AI 在過長的上下文中失去焦點，保持回應的準確性

建立記憶

在專案根目錄建立 CLAUDE.md 檔案，記錄專案規範、常用命令和程式碼風格，讓 AI 更了解你的需求

🛠️ 快速設定 MCP 工具

設定 MCP 工具就像安裝手機 App 一樣簡單。以下示範如何連接檔案系統工具， 讓 Claude Code 能夠幫你整理和管理專案檔案。

# 添加檔案系統 MCP 伺服器
claude mcp add filesystem npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/files
# 檢視已連接的 MCP 伺服器
claude mcp list
# 在對話中檢查 MCP 狀態
/mcp

專業提示：建議從檔案系統 MCP 開始體驗，它能讓 Claude Code 幫你整理下載資料夾、 分析專案結構，甚至協助重構程式碼檔案，是最實用的入門工具。
