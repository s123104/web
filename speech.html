章節導航

學業壓力與憂鬱風險之相關性分析

總時間：15分鐘 | Q&A：2-3分鐘

1

封面

10秒

開場白： 各位老師、各位同學，大家好！今天我們要報告的是「學業壓力與憂鬱風險之相關性分析」。

2-3

摘要

1分鐘

核心發現： 我們分析了近 2.8 萬名大學生問卷資料，證實「學業壓力越大，憂鬱風險越高」。這兩者呈中度正相關 (r = 0.475)。

數據支持： 你看，壓力低的時候只有 19.44% 的人有憂鬱風險，但壓力高時，這個比例飆升到 81.63%！這個差異非常顯著。

研究方法： 我們用了描述性統計、卡方檢定，以及兩種機器學習模型來預測（Logistic Regression 準確率 73.4%，AUC 0.805；Random Forest 準確率 69.3%，AUC 0.750）。

實用工具： 我們還整合了 Python、MySQL 和 Grafana，建立了一套動態監控系統，學校可以即時看到高風險學生。

最終建議： 根據分析結果，我們建議學校針對高壓力學生提供相關協助。

4-6

緒論

0.5分鐘

研究背景與動機： 隨著學業競爭加劇，大學生面臨的壓力日益增加，這對心理健康可能產生深遠影響，尤其是憂鬱風險。台灣目前對這塊的實證研究還不夠。

研究目的： 本研究旨在探討學業壓力與憂鬱風險之間的關聯性，並建立預測模型，最終提供一套資料視覺化整合系統，為學校提供早期預警機制與介入策略。

7-10

文獻回顧

0.5分鐘

過去研究： 學術壓力、課業負擔、考試焦慮等因素已被證明與學生心理健康問題相關。

本研究貢獻： 我們結合了多面向資料，使用機器學習模型，並建立即時監控系統，希望能填補這塊的空白。

11-15

研究方法 - 資料來源與樣本

1.5分鐘

資料來源： 我們的資料來自 Kaggle 的匿名大學生問卷，涵蓋了學業壓力、憂鬱風險等關鍵變項。

資料清洗： 我們做了嚴謹的資料清洗，包括缺失值處理、重複值刪除、離群值處理，並將分類變數進行了數值化轉換。最終樣本數是 27,873 筆。

特徵工程： 我們將學業壓力進行了分級，並對學歷、性別等進行了 One-hot 編碼，讓資料更能被模型理解。

16-18

描述性統計與相關性分析

1.5分鐘

基本資料： (快速帶過數據表格，強調樣本數量和主要變項的分布)

壓力與憂鬱比例： 我們發現學業壓力等級越高，憂鬱風險的比例就越高。

低壓力組

19.44%

高壓力組

81.63%

相關性分析： 學業壓力與憂鬱風險呈中度正相關 (Pearson r = 0.475)。卡方檢定也顯示兩者間存在顯著差異 (p < 0.001)，這代表壓力確實會顯著影響憂鬱風險。

19-20

機器學習模型

0.5分鐘

模型選擇： 我們選用了兩種常用的分類模型：Logistic Regression (羅吉斯迴歸) 和 Random Forest (隨機森林)。

模型訓練： 資料經過標準化處理後，我們以 80% 作為訓練集、20% 作為測試集進行模型訓練。

模型評估： 我們主要關注準確率 (Accuracy)、精確度 (Precision)、召回率 (Recall) 和 AUC (曲線下面積)。

21-26

混淆矩陣與模型評估

3分鐘

混淆矩陣是什麼？ 它是一個表格，用來視覺化分類模型在測試集上的表現，告訴我們模型預測的「有」或「沒有」跟真實情況是不是一致。

True Positive (TP)

真正有憂鬱，模型也預測有憂鬱

True Negative (TN)

真正沒有憂鬱，模型也預測沒有憂鬱

False Positive (FP)

真正沒有憂鬱，模型卻預測有憂鬱（偽陽性）

False Negative (FN)

真正有憂鬱，模型卻預測沒有憂鬱（偽陰性）

精確度 (Accuracy) 和 準確率 (Precision) 的差異：

  • Accuracy (整體準確率)： 指的是模型整體預測正確的比例，計算公式是 (TP + TN) / (TP + TN + FP + FN)。
  • Precision (精確度)： 衡量的是在所有模型預測為正的樣本中，有多少是真正為正的。計算公式是 TP / (TP + FP)。

模型表現數值解釋：

Precision(有憂鬱)

0.80

預測有憂鬱中，80%確實有

Recall(有憂鬱)

0.73

真正有憂鬱中，73%被找出

Precision(無憂鬱)

0.66

預測無憂鬱中，66%確實無

Recall(無憂鬱)

0.74

真正無憂鬱中，74%被找出

27-28

特徵重要性分析

1分鐘

誰是關鍵？ 學業壓力在兩個模型中都顯示出最⾼的重要性， 在 Logistic Regression 與 Random Forest 均為 Rank #1， 再次證明它是預測憂鬱風險的關鍵因素。

還有誰重要？ 年齡、CGPA (學業平均成績)、學習滿意度等也對憂鬱風險有影響。

29-32

ROC曲線與AUC值

0.5分鐘

ROC曲線： 這條曲線視覺化呈現了分類模型在不同判斷門檻下的表現。曲線越靠近左上角越好，代表模型分類效果越棒。

AUC值： 曲線下面積，衡量模型區分「有憂鬱」和「無憂鬱」這兩類的能力。AUC 值越高代表模型分類能力越好。我們的模型 AUC 值都在 0.75 以上，表現良好。

33-37

Random Forest 原理 - 隨機森林簡單講

1.5分鐘

隨機森林模型是怎麼預測的？

它就像一個「大老闆」，不像一個單打獨鬥的「小員工」。它不只用一棵決策樹來判斷，而是訓練了很多很多棵「隨機」的小決策樹，然後讓這些樹各自做出判斷，最後再集合大家的意見，用「投票」的方式來決定最終結果。

怎麼做到「隨機」？

  • 1
    資料隨機抽樣： 每棵小樹訓練用的資料都是從原始資料中隨機抽取出來的，而且可以重複抽（這叫做 Bootstrap Sampling）。
  • 2
    特徵隨機選擇： 建立每棵樹的時候，它不是看所有特徵來分裂，而是只從隨機選取的一小部分特徵中挑選。

這樣有什麼好處？ 因為每棵小樹都是獨立思考的，而且有隨機性，所以它們的判斷會比較多樣化。當大家一起投票時，就能大大減少單一決策樹可能犯的錯誤，讓最終的預測結果更穩定、更準確，也比較不會過度擬合（Overfitting）。

38-43

資料庫整合與視覺化

2分鐘

我們怎麼讓數據活起來？ 我們建立了一個「三層樓」系統：

資料前處理層 (Python)： 這是我們的「廚房」，負責資料清洗、特徵工程、模型訓練和預測結果產出。

模組化設計： 我們把整個分析流程拆分成一個個獨立的、可以重複使用的「積木」或「零件」。例如，資料清洗是一個模組，模型訓練是另一個模組。這樣當有新資料進來時，只需要針對特定的「積木」進行更新，讓整個系統更靈活、好維護。

資料持久層 (MySQL 資料庫)： 這是我們的「倉庫」，把 Python 分析結果全部存起來，方便管理和查詢。它也支援索引優化，確保視覺化時資料查詢速度快。

視覺化展示層 (Grafana 儀表板)： 這是我們的「展示廳」，透過連接 MySQL，把數據變成即時更新的互動式圖表，超直觀！

資料流轉： 整個系統是單向資料流設計，確保分析結果的一致性與可追溯性。Python 處理完資料後寫入 MySQL，Grafana 再從 MySQL 讀取並顯示。

44-46

實際貢獻與價值

1分鐘

這個系統對學校有什麼用？

及早預警

透過數據監控，學校可以更早發現那些面臨高學業壓力的學生，有效預防憂鬱風險。

精準干預

根據我們詳細的數據分析結果，學校可以制定更具針對性的心理健康輔導策略，例如針對不同學歷或性別的學生提供差異化的幫助。

資源優化

幫助學校合理分配心理健康資源，提升效益，讓資源用在最需要的地方。

47

結論與建議

0.5分鐘

總結： 學業壓力確實是影響大學生憂鬱風險的重要因素。我們的預測模型和資料視覺化系統，能夠有效協助學校進行監測和干預。

我們的建議： 學校應該加強對高學業壓力學生的心理健康支持，例如開設壓力管理課程、提供個別諮詢服務，並持續追蹤數據變化。未來研究可以考慮納入睡眠和社交支持等更多變項，建立更全面的長期追蹤系統。
