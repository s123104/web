Docker 環境設置

為了更好地使用這些 Docker 指令，請輸入你的 Docker 專案路徑

● 官方推薦路徑結構
Windows:
C:\Users\使用者名稱\docker-projects
macOS:
~/docker-projects
等同於 /Users/使用者名稱/docker-projects
Linux:
~/docker-projects
等同於 /home/使用者名稱/docker-projects
自動整合範例

輸入上方路徑後，以下指令會自動套用：

1. 開啟 VS Code
2. 終端機中輸入: cd 你的Docker路徑
3. 建立資料夾: mkdir -p data
4. 啟動服務: docker-compose up -d

所有 volume 掛載路徑將自動從 ./data 調整為您設定的路徑

啟動與結束

啟動服務（背景執行）

在背景模式下啟動容器，執行後可繼續使用終端機。 

docker-compose up -d
-d 代表 detached mode，讓容器在背景執行，最常用來開始專案。
啟動與結束

關閉所有服務

停止並移除由 docker-compose 啟動的所有容器。 

docker-compose down
停止並移除容器，但保留資料卷(volume)，資料會保存。
啟動與結束

關閉並刪除資料

停止容器並清除所有相關資源，包括資料卷和網路。 

docker-compose down -v
會清除所有資料，請謹慎使用。適合完全重置環境。
啟動與結束

重新啟動服務

重新啟動所有或特定的容器，通常用於套用配置變更。 

docker-compose restart
也可以指定特定服務：docker-compose restart 服務名稱，不會重建容器。
啟動與結束

重新建置後啟動

強制重新建置映像檔後啟動容器，適用於修改 Dockerfile 後。 

docker-compose up --build -d
當修改 Dockerfile 或依賴變更時使用，能夠更新程式碼。
檢查與監控

查看執行中容器

列出當前正在運行的所有 Docker 容器。 

docker ps
顯示容器 ID、名稱、狀態等資訊，方便查看運行狀態。
檢查與監控

查看所有容器

顯示所有容器，包括已停止的，可清楚了解系統狀態。 

docker ps -a
-a 參數顯示所有容器，包括已停止的，提供完整容器列表。
檢查與監控

查看即時日誌

持續監控所有服務的日誌輸出，適合即時偵錯。 

docker-compose logs -f
-f 參數實時追蹤日誌更新，可按 Ctrl+C 中斷，方便即時觀察輸出。
檢查與監控

查看特定服務日誌

只監控特定服務的日誌，減少干擾雜訊，專注除錯。 

docker-compose logs service-name
將 service-name 替換為您要查看的具體服務名稱，例如 logs memory。
檢查與監控

查看端口映射

顯示所有服務的容器及其端口映射關係。 

docker-compose ps
列出所有服務及本機端口到容器端口的映射，方便確認端口綁定。
清理與資源

停止特定容器

停止指定的容器，保留容器但停止其執行。 

docker stop container_id
使用 docker ps 獲取容器 ID，暫停運行但不刪除容器。
清理與資源

移除容器

永久刪除指定的容器，釋放系統資源。

docker rm container_id
容器必須先停止才能刪除，除非使用 -f 強制刪除，不會刪除映像檔。
清理與資源

刪除映像檔

刪除指定的 Docker 映像檔，適合釋放磁碟空間。 

docker rmi image_id
使用 docker images 獲取映像檔 ID，可以節省硬碟空間。
清理與資源

清理未使用資源

移除所有未使用的容器、網路、映像檔和快取。 

docker system prune
會移除所有停止的容器和未使用的網路，建議定期執行維護。
清理與資源

全系統重置

徹底清理 Docker 環境，包括所有容器、映像檔和資料卷。 

docker system prune -a --volumes
危險操作！將刪除所有 Docker 資源，包括資料卷，僅緊急時使用。
進入與除錯

進入容器 Shell

連接到容器的 Shell 環境，可直接在容器內執行命令。 

docker exec -it container_id /bin/sh
多數 Linux 容器使用 /bin/sh，使用 docker ps 獲取容器 ID。
進入與除錯

進入 Bash Shell

連接到容器的 Bash Shell（如果支援），提供更豐富的功能。 

docker exec -it container_id bash
部分容器可能沒有 bash，這時可以用 /bin/sh，功能更完整。
進入與除錯

檢查容器資訊

獲取容器的詳細信息，包括網絡配置、IP 地址等。 

docker inspect container_id
返回完整的 JSON 格式容器配置與狀態，適合排除網路問題。
進入與除錯

檢查容器掛載點

查看容器的 volume 掛載情況，解決資料持久化問題。 

docker inspect -f '{{ .Mounts }}' container_id
使用格式化輸出只顯示掛載點資訊，資料問題必查。
進階操作

重建特定服務

重新建置並啟動特定服務，節省時間，只更新需要的部分。 

docker-compose up service-name --build -d
替換 service-name 為要重建的服務名稱，適合單服務更新。
進階操作

啟動指定服務

只啟動配置文件中的特定服務，而非全部啟動。 

docker-compose up memory brave-search -d
可同時指定多個服務，以空格分隔，按需啟動。
進階操作

檢查配置

顯示當前環境的完整 docker-compose 配置，查找錯誤。 

docker-compose config
可查看環境變數替換後的最終配置，適合配置檢查。
Claude MCP

啟動記憶服務

啟動 Claude MCP 記憶模組，讓 AI 記住對話內容與文件。 

docker-compose up memory -d
對應 @modelcontextprotocol/server-memory 套件，長期記憶必備。
Claude MCP

啟動搜尋功能

啟動 Brave Search 服務，讓 Claude 能夠搜尋網路資訊。 

docker-compose up brave-search -d
對應 @modelcontextprotocol/server-brave-search 套件，能連網查資料。
Claude MCP

啟動檔案系統服務

啟動專用檔案系統服務，讓 Claude 能針對特定資料夾進行讀寫操作。 

docker-compose up filesystem -d
對應 @modelcontextprotocol/server-filesystem 套件，比 everything 更專注於檔案操作。
Claude MCP

啟動通用檔案存取

啟動檔案系統存取，讓 Claude 能讀寫本地檔案，並提供更多功能。 

docker-compose up everything -d
對應 @modelcontextprotocol/server-everything 套件，提供更全面的功能。
Claude MCP

啟動 Git 功能

啟動 Git 操作服務，讓 Claude 能夠進行版本控制操作。 

docker-compose up git -d
對應 @modelcontextprotocol/server-git 套件，實現版本控制整合。
進階操作

資料保存術

設定本地資料夾映射，實現 Claude 讀寫本地檔案的能力。 

volumes:
  - ./data:/app/data
加入此設定到 docker-compose.yml 的服務配置中，實現資料永久保存。
