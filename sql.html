2. 資料庫、欄位與資料型態大解密

狼：一個資料庫就像超大倉庫，裡面有許多「表格」當儲物區， 而欄位就像儲物格，能放各種型態的寶物 (灑花✧٩(ˊᗜˋ*)و✧)。 常見的大分類如下：

  1. 數值資料 (Numeric Data)：例如 integer、float、money 等，適合存放數字、金額、價格等等唷！
  2. 字串資料 (Character & Strings Data)：可以存放各種文字、符號，小兔的日記、名字都能放進去。
  3. 日期/時間資料 (Date/Time Data)：用來記錄日期或時間（如 Date、Time、Timestamp）。
  4. 布林值 (Boolean Data)：True/False、Yes/No、1/0， 二選一簡單明瞭。
  5. 二進位字串 (Binary Data)：可以放圖片或檔案等二進位內容，但有時也會只存路徑，以免占用空間。

小兔要注意唷，不同資料庫或版本，型態與名稱都有些微差異，但大致概念類似♪

2.3 狼兔劇場 - 資料庫關聯 (一對多、多對一、多對多)

狼：咿呀，小兔，我們的資料表不只是各自獨立，有時候還要互相「關聯」呢～ 兔：哇，聽起來就像一群好朋友彼此都有連結，真有趣 (*´▽`*)！

其實關聯可以幫助我們保持資料的一致性和彈性。假設有 「狼表 (wolves)」 和 「兔表 (rabbits)」， 以及一張中間表來記錄誰喜歡誰，或者誰被誰追捕等等XD

  1. 一對多 (One-to-Many)： 一隻狼可以擁有好多兔子（像是狼對兔都充滿興趣(❁´ω`❁)）， 在資料表裡就是「狼表」的主鍵 (wolf_id) 會放在「兔表」裡作為外鍵。
  2. 多對一 (Many-to-One)： 和「一對多」是相同概念，只是從另一邊看，一堆兔子都歸屬同一隻狼～ 在「兔表」(rabbits) 裡存 wolf_id 指向「狼表」(wolves) 的 id。
  3. 多對多 (Many-to-Many)： 像是狼群可以同時注意好多兔子，兔子也可以被多隻狼追(>_<) 這種情況就需要一張「中介表 (junction table)」紀錄雙方 id。

狼：透過這些關聯，我們就能更輕鬆地查詢、管理和維護資料。例如：

-- 建立「狼表」
CREATE TABLE wolves (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50)
);

-- 建立「兔表」，用 wolf_id 當外鍵
CREATE TABLE rabbits (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50),
  wolf_id INT,
  FOREIGN KEY (wolf_id) REFERENCES wolves(id)
);

兔：好棒！這樣就能知道每隻兔子跟哪隻狼有關係(๑´ㅂ`๑)♡*.+，還能保障資料正確哩♪

3. MySQL 速成教學

狼：MySQL 就像一個超大型冰箱，可容納各種食材（資料）。只要選對資料型態，管理就輕鬆愉快！

3.1 連線資料庫 🔓 (Open the Fridge)

mysql -u root -p

以 root 管理員登入，輸入密碼後就能開冰箱囉～

3.2 創建資料庫 🏠 (Set Up the Kitchen)

CREATE DATABASE Kitchen;
USE Kitchen;

建立名為 Kitchen 的分區，準備擺放資料表～

3.3 建立資料表 🗄️ (Install Shelves)

CREATE TABLE Ingredients (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50),
    quantity INT,
    expiry DATE
);

在 Kitchen 裡創建 Ingredients，像在冰箱中裝好層架，隨時可以放食物(〃∀〃)

3.4 新增資料 ➕ (Insert Items)

INSERT INTO Ingredients (name, quantity, expiry)
VALUES ('Eggs', 12, '2025-02-28');

丟進 12 顆雞蛋，並設定保存期限，真貼心～

3.5 查詢資料 🔍 (Check Inventory)

SELECT * FROM Ingredients;

打開冰箱，看看現在有什麼食材♪

3.6 更新資料 ✏️ (Adjust Stock)

UPDATE Ingredients
SET quantity = 10
WHERE name = 'Eggs';

把雞蛋數量從 12 改成 10，或許我們用掉了一些？(๑´ㅂ`๑)

3.7 刪除資料 🗑️ (Discard Expired Items)

DELETE FROM Ingredients
WHERE name = 'Eggs';

把過期食材丟掉～但要小心條件，別一次全砍光囉(⊙△⊙)

3.8 依照到期日排序資料 (ORDER BY)

SELECT *
FROM Ingredients
ORDER BY expiry ASC;

先找出快到期的食材來用，非常省時不浪費 (°∀°)b

想 ASC 升冪還是 DESC 降冪排序都行ヽ(・∀・)ﾉ

3.9 刪除資料庫 / 清除內容

※ 警告：以下動作會讓資料「再見不回頭」(ಥ﹏ಥ)，請謹慎使用！

  • 整個資料庫都砍掉 (Drop Database)
    DROP DATABASE Kitchen;
    這條下去後，整個 Kitchen（連同裡頭所有表格與資料）都會不見囉。

  • 只清除某張表的內容 (Truncate/Delete)
    TRUNCATE TABLE Ingredients; -- 一次咻地全清光，ID歸零
    DELETE FROM Ingredients;      -- 一筆一筆刪掉，但自動增值計數器保留
    有時候用於測試資料或清空表格，可不要一時手快把正式資料炸掉呀(>ω<)！

    • 3.10 PHP + MySQL 操作範例 (常見增刪改查)

    以下以 mysqli 為例♪

    <?php
// 連線參數
$host = 'localhost';
$user = 'root';
$pass = '你的密碼';
$db   = 'Kitchen';

// 建立連線
$conn = mysqli_connect($host, $user, $pass, $db);
if (!$conn) {
    die('無法連線: ' . mysqli_connect_error());
}

// 1) 新增 INSERT
$sql_insert = "INSERT INTO Ingredients (name, quantity, expiry) 
               VALUES ('Sugar', 1, '2025-12-31')";
mysqli_query($conn, $sql_insert);

// 2) 查詢 SELECT
$sql_select = "SELECT * FROM Ingredients";
$result = mysqli_query($conn, $sql_select);
while ($row = mysqli_fetch_assoc($result)) {
    echo "ID: ".$row['id']." / 名稱: ".$row['name']." / 數量: ".$row['quantity']." / 到期日: ".$row['expiry']."
    ";
}

// 3) 更新 UPDATE
$sql_update = "UPDATE Ingredients 
               SET quantity = 2 
               WHERE name = 'Sugar'";
mysqli_query($conn, $sql_update);

// 4) 刪除 DELETE
$sql_delete = "DELETE FROM Ingredients 
               WHERE name = 'Sugar'";
mysqli_query($conn, $sql_delete);

// 關閉連線
mysqli_close($conn);
?>

GO
USE Kitchen;
