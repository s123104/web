專案介紹 準備工作 材料清單 電路組裝 程式碼詳解 測試與除錯

🚀 專案介紹：打造你的第一個智慧保全！

歡迎來到 Arduino 的奇妙世界！在這個教學中，我們將一步步帶你使用常見的電子元件，打造一個簡易但有效的「光控防盜警報器」。 這個系統的原理很簡單：利用光敏電阻偵測環境光線變化，當有異常光線（例如，原本黑暗的抽屜被打開照到光）時，系統就會觸發 LED 燈閃爍並讓蜂鳴器發出聲響，達到警示效果。

這個專案非常適合 Arduino 初學者，可以讓你熟悉：

  • Arduino 基本程式結構
  • 數位訊號輸出 (控制 LED 和蜂鳴器)
  • 類比訊號讀取 (讀取光敏電阻數值)
  • 基本的電路連接

🛠️ 準備工作：軟硬體設定

1. 認識 Arduino 主控板

Arduino 是一塊開源的微控制器板，你可以把它想像成一個迷你電腦，可以透過寫程式來控制連接到它的各種電子元件（感測器、馬達、燈光等）。最常見的入門板是 Arduino UNO。

[這裡可以放一張 Arduino UNO 板的圖片]

2. 安裝 Arduino IDE (整合開發環境)

我們需要在電腦上安裝 Arduino IDE 這個軟體來編寫、上傳程式碼到 Arduino 板子上。

  1. 前往 Arduino 官方網站下載頁面：https://www.arduino.cc/en/software
  2. 根據你的作業系統 (Windows, macOS, Linux) 下載最新版本的 IDE。
  3. 下載完成後，依照安裝程式的指示完成安裝。
  4. 安裝過程中可能會提示安裝驅動程式，請允許安裝。

3. 連接 Arduino 並設定 IDE

  1. 使用 USB 線將你的 Arduino 板連接到電腦。
  2. 打開 Arduino IDE 軟體。
  3. 在 IDE 的選單列，選擇「工具 (Tools)」>「開發板 (Board)」，然後選擇你使用的 Arduino 型號（例如 "Arduino UNO"）。
  4. 再選擇「工具 (Tools)」>「序列埠 (Port)」，選擇 Arduino 連接到的那個序列埠（通常會標示 Arduino 的名稱）。如果有多個選項不確定，可以先拔掉 Arduino 再看哪個選項消失，那個就是了。

設定完成後，你的電腦和 Arduino 就準備好溝通了！

🔌 電路組裝：連接你的元件

現在我們要將所有元件透過麵包板和杜邦線連接起來。請仔細按照以下步驟操作，確保接線正確。

[這裡強烈建議放一張清晰的 Fritzing 電路圖 或 實際接線照片]

接線步驟：

  1. 連接光敏電阻 (LDR) 分壓電路：
    • 將 10kΩ 電阻的一端連接到 Arduino 的 5V 腳位。
    • 將 10kΩ 電阻的另一端連接到麵包板的某一排。
    • 將光敏電阻的一端連接到麵包板上與 10kΩ 電阻相連的那一排。
    • 將光敏電阻的另一端連接到 Arduino 的 GND (接地) 腳位。
    • **重要：** 從 10kΩ 電阻和光敏電阻連接的那個節點（麵包板上的同一排）拉一條線，連接到 Arduino 的 A0 (類比輸入) 腳位。
    • (原理：這形成了一個分壓電路。光線越強，LDR 電阻越小，A0 讀到的電壓就越低。)
  2. 連接 LED：
    • 將 220Ω 電阻的一端連接到 Arduino 的數位腳位 D2。
    • 將 220Ω 電阻的另一端連接到 LED 的長腳（正極，陽極）。
    • 將 LED 的短腳（負極，陰極）連接到 Arduino 的 GND 腳位。
  3. 連接蜂鳴器：
    • 找到蜂鳴器的正極（通常標示 '+' 或比較長）。將正極連接到 Arduino 的數位腳位 D3。
    • 將蜂鳴器的負極（通常標示 '-' 或比較短）連接到 Arduino 的 GND 腳位。
    • (如果使用無源蜂鳴器，接法相同，但需要用 `tone()` 函數驅動。)
  4. 檢查： 仔細檢查所有接線是否牢固、是否插對腳位，特別是 5V 和 GND 不要接反。

💻 程式碼詳解：賦予系統靈魂

電路連接好後，我們需要編寫程式碼告訴 Arduino 如何運作。打開 Arduino IDE，複製並貼上以下程式碼： 


// 定義腳位
const int ldrPin = A0;     // 光敏電阻連接到類比腳位 A0
const int ledPin = 2;      // LED 連接到數位腳位 D2
const int buzzerPin = 3;   // 蜂鳴器連接到數位腳位 D3

// 定義光線觸發閾值 (需要根據實際環境調整)
// 在我們的接線中，光線越強，讀值越低
int lightThreshold = 400; // 低於此值表示偵測到足夠光線 (觸發警報)

void setup() {
  // 初始化序列埠通訊，方便除錯 (觀察數值)
  Serial.begin(9600);

  // 設定 LED 和蜂鳴器腳位為輸出模式
  pinMode(ledPin, OUTPUT);
  pinMode(buzzerPin, OUTPUT);

  // 光敏電阻腳位 (A0) 預設就是輸入模式，可省略 pinMode(ldrPin, INPUT);

  // 初始狀態：關閉 LED 和蜂鳴器
  digitalWrite(ledPin, LOW);
  digitalWrite(buzzerPin, LOW);

  Serial.println("光控防盜系統啟動...");
}

void loop() {
  // 讀取光敏電阻的類比數值 (範圍 0-1023)
  int ldrValue = analogRead(ldrPin);

  // 將讀取的數值透過序列埠傳送到電腦，方便觀察和調整閾值
  Serial.print("光線感測值: ");
  Serial.println(ldrValue);

  // 判斷光線是否達到觸發條件
  if (ldrValue < lightThreshold) {
    // 光線充足 (低於閾值)，觸發警報
    Serial.println("偵測到光線！觸發警報！");
    digitalWrite(ledPin, HIGH);    // 開啟 LED
    digitalWrite(buzzerPin, HIGH);   // 開啟蜂鳴器 (適用於有源蜂鳴器)
    // 如果是無源蜂鳴器，使用 tone(buzzerPin, 1000); // 發出 1000 Hz 聲音
  } else {
    // 光線不足 (高於或等於閾值)，解除警報
    digitalWrite(ledPin, LOW);     // 關閉 LED
    digitalWrite(buzzerPin, LOW);    // 關閉蜂鳴器
    // 如果是無源蜂鳴器，使用 noTone(buzzerPin);
  }

  // 稍微延遲，避免讀取過快或洗頻
  delay(200); // 延遲 200 毫秒
}

程式碼說明：

  • 變數定義： 我們先定義了各個元件連接的腳位 (ldrPin, ledPin, buzzerPin) 和觸發警報的光線閾值 (lightThreshold)。這個閾值需要你根據實際環境亮度來調整。
  • setup() 函數： 這個函數只在 Arduino 啟動時執行一次。我們在這裡設定 LED 和蜂鳴器的腳位為 `OUTPUT`（輸出模式），並啟動序列埠通訊 (`Serial.begin(9600)`)，以便在電腦上看到感測器的讀值。
  • loop() 函數： 這個函數會不斷重複執行。
    • analogRead(ldrPin)：讀取 A0 腳位的類比電壓值，範圍是 0 到 1023。由於我們的接法，光線越強，讀值越低。
    • Serial.println(ldrValue)：將讀取到的數值顯示在 Arduino IDE 的「序列埠監控視窗」中。
    • if (ldrValue < lightThreshold)：這是核心判斷。如果讀取的數值 *小於* 我們設定的閾值（表示光線夠強），就執行 `if` 區塊內的程式碼。
    • digitalWrite(ledPin, HIGH) / digitalWrite(buzzerPin, HIGH)：將對應的數位腳位設為高電位 (HIGH)，點亮 LED、啟動蜂鳴器。
    • else：如果讀取的數值 *不小於* 閾值（表示光線較暗），就執行 `else` 區塊內的程式碼。
    • digitalWrite(ledPin, LOW) / digitalWrite(buzzerPin, LOW)：將腳位設為低電位 (LOW)，關閉 LED 和蜂鳴器。
    • delay(200)：暫停 200 毫秒，讓系統穩定一下，也方便觀察序列埠的輸出。

重要：調整閾值 (Threshold)

lightThreshold 的值 (預設 400) 非常關鍵。你需要：

  1. 上傳程式碼到 Arduino。
  2. 打開 Arduino IDE 的「工具 (Tools)」>「序列埠監控視窗 (Serial Monitor)」（確保右下角波特率設為 9600）。
  3. 觀察在「正常黑暗」狀態下，序列埠輸出的數值大概是多少。
  4. 再觀察用手電筒照射或打開抽屜時，「偵測到光線」狀態下，數值又是多少。
  5. 選擇一個介於這兩種狀態之間的數值作為你的 `lightThreshold`。例如，如果黑暗時讀值約 700-800，有光時讀值約 100-200，那麼 400 是一個不錯的起始閾值。
  6. 修改程式碼中的 `lightThreshold` 值，然後重新上傳程式碼。

🔬 測試與除錯：讓系統跑起來！

  1. 再次檢查電路接線是否正確無誤。
  2. 將程式碼上傳到 Arduino 板（點擊 IDE 上方的 "→" 上傳按鈕）。
  3. 打開序列埠監控視窗 (Tools > Serial Monitor)。你應該會看到不斷輸出的光線感測值。
  4. 用手遮住光敏電阻，觀察數值變化，以及 LED 和蜂鳴器是否熄滅。
  5. 移開手或用手電筒照射光敏電阻，觀察數值變化，以及 LED 和蜂鳴器是否啟動。
  6. 根據觀察結果，回到程式碼調整 `lightThreshold` 值，直到系統反應符合你的預期。

遇到問題？

  • 程式碼無法上傳？ 檢查是否選對開發板和序列埠。確認 USB 線連接良好。
  • 序列埠沒反應？ 檢查 `Serial.begin(9600)` 是否正確？監控視窗右下角波特率是否也是 9600？
  • LED 不亮？ 檢查 LED 正負極是否接反？限流電阻是否連接正確？D2 腳位是否接對？GND 是否接好？
  • 蜂鳴器不響？ 檢查蜂鳴器正負極是否接反（有源蜂鳴器）？D3 腳位是否接對？GND 是否接好？
  • 感測值一直不變或數值異常？ 檢查 A0、5V、GND 的接線是否正確牢固。光敏電阻或 10kΩ 電阻是否接觸不良？
  • 警報觸發時機不對？ 主要問題在於 `lightThreshold`，請依照上面「調整閾值」的步驟操作。
