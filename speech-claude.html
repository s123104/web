學業壓力與憂鬱風險之相關性分析

總時間：18分鐘 | Q&A：2-3分鐘

1

封面

10秒

開場白： 各位老師、各位同學，大家好！今天我們要報告的是「學業壓力與憂鬱風險之相關性分析」。

2-3

摘要

1分鐘

核心發現： 我們分析了近 2.8 萬名大學生問卷資料，證實「學業壓力越大，憂鬱風險越高」。這兩者呈中度正相關 (r = 0.475)。

數據支持： 你看，壓力低的時候只有 19.44% 的人有憂鬱風險，但壓力高時，這個比例飆升到 81.63%！這個差異非常顯著。

研究方法： 我們用了描述性統計、卡方檢定，以及兩種機器學習模型來預測（Logistic Regression 準確率 73.4%，AUC 0.805；Random Forest 準確率 69.3%，AUC 0.750）。

實用工具： 我們還整合了 Python、MySQL 和 Grafana，建立了一套動態監控系統，學校可以即時看到高風險學生。

最終建議： 根據分析結果，我們建議學校針對高壓力學生提供相關協助。

4

緒論

0.3分鐘

研究背景與動機： 隨著學業競爭加劇，大學生面臨的壓力日益增加，這對心理健康可能產生深遠影響，尤其是憂鬱風險。台灣目前對這塊的實證研究還不夠。

研究動機

0.2分鐘

為什麼要做這個研究？ 現在大學生面臨越來越大的學業壓力，但我們缺乏具體的數據來了解這種壓力究竟對心理健康有多大影響。我們希望用數據說話，讓學校能夠更有針對性地幫助學生。

PCA 分析:PC1 解釋變異比例與前五大載荷量

1分鐘

PCA 是什麼？ PCA 就像是「資料壓縮大師」，它能把很多變數濃縮成幾個主要成分，幫我們找出最重要的資訊。想像一下，如果你有很多照片要存到手機裡，PCA 就是幫你把照片壓縮但保留最重要內容的工具。

PC1 解釋變異比例： 第一主成分 (PC1) 能解釋大約 45% 的資料變異。這表示我們資料中將近一半的資訊都可以用這一個成分來代表，這是很不錯的結果！

前五大載荷量： 在第一主成分中，學業壓力、憂鬱狀況、學習滿意度、CGPA 和年齡是載荷量最高的變數。這告訴我們，這些變數在解釋學生心理健康狀況時是最重要的因子。

白話解釋： 就像是在一個班級裡，如果要用一個指標來代表學生的整體狀況，學業壓力和憂鬱情況會是最能代表這個班級特色的指標。

Random Forest:模型介紹與準確率

1.5分鐘

隨機森林模型是怎麼預測的？

它就像一個「大老闆」，不像一個單打獨鬥的「小員工」。它不只用一棵決策樹來判斷，而是訓練了很多很多棵「隨機」的小決策樹，然後讓這些樹各自做出判斷，最後再集合大家的意見，用「投票」的方式來決定最終結果。

怎麼做到「隨機」？

  • 1
    資料隨機抽樣： 每棵小樹訓練用的資料都是從原始資料中隨機抽取出來的，而且可以重複抽（這叫做 Bootstrap Sampling）。
  • 2
    特徵隨機選擇： 建立每棵樹的時候，它不是看所有特徵來分裂，而是只從隨機選取的一小部分特徵中挑選。

Random Forest 表現： 我們的 Random Forest 模型準確率為 69.3%，AUC 值為 0.750。雖然準確率比 Logistic Regression 稍低，但它提供了更好的特徵重要性分析。

還有誰重要？ 年齡、CGPA (學業平均成績)、學習滿意度等也對憂鬱風險有影響。

資料流轉與自動化:ORM 實現與自動化腳本流程

2分鐘

什麼是自動化？簡單說就是「按一個按鈕，一切搞定」！

想像一下這個情況： 以前如果有新的學生問卷資料進來，可能需要一個工程師花好幾個小時來處理：先清理資料、再跑分析、然後手動把結果輸入到資料庫、最後更新圖表。這樣既累又容易出錯。

現在有了自動化系統： 只要把新資料丟進去，系統就會自動開始工作！就像洗衣機一樣，你只要按個開始按鈕，它就會自動洗、脫水、烘乾，全部搞定。 

# 這是我們的「一鍵搞定」程式
# 1. 自動建立資料庫連線（就像插插頭）
engine = create_engine("mysql+pymysql://...")

# 2. 自動檢查資料庫存不存在，不存在就建一個（就像檢查有沒有洗衣籃）
with open('create_table.sql', 'r') as sql_file:
    conn.execute(text(sql_file.read()))

# 3. 自動把 27,873 筆資料分批寫入（就像分批洗衣服）
df.to_sql('student_depression', con=engine, 
          if_exists='replace', chunksize=1000)

ORM 是什麼？ ORM 就像是「翻譯員」。電腦理解的是 0 和 1，資料庫說的是 SQL 語言，但我們寫程式用的是 Python。ORM 就是那個會說三種語言的翻譯員，讓大家都能溝通！
